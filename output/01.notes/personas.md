# Personas

## User A: 
I am familiar with Terraform, already have an HPC TF (or TFE?) account but new to Azure. I was browsing through services on Az portal to see if there was an easy way for me to manage Azure infrastructure. I want to use the familiar terraform experience, my existing credentials and my tf skills to manage Azure infrastructure without coming out of the Azure portal. This way I feel I can monitor and confirm my terraform apply on az portal without changing context. 

## User B: 
I am new to Terraform but I have managed Azure infrastructure for some time. I have been on, and have used, Azure portal (ARM) to do most of my services and resources provisioning so far.  I want to expand on that deployment and want to bring in other teams for management of those resources. I want to use terraform for better management and governance of those workspaces. 

## User C: 
I am new to both Terraform and Azure. I may be coming from an on-prem data center or being on one cloud (e.g., AWS). I have been told to expand and start using Azure. I need to port some of my services to start using Azure infrastructure and want a consistent way that I can manage my infrastructure between Azure and my existing footprint with common tools and I feel using tf will enable me to do that.
