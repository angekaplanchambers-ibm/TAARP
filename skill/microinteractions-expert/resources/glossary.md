# Microinteraction Glossary

## Terms

- **Trigger**: Event that starts the interaction.
- **Precondition**: Requirement that must be true before the interaction can proceed.
- **Feedback**: User-visible response to a trigger.
- **State transition**: Change from one interaction state to another.
- **Recovery path**: User action that resolves an error or blocked state.
- **Timing budget**: Acceptable response time for each interaction phase.
- **Guardrail metric**: Metric that detects regressions or harmful outcomes.
- **Reduced motion mode**: Alternative behavior that minimizes animation.

## Severity Bands (for critique)

- **Critical**: Blocks core task, causes data loss risk, or major accessibility failure.
- **Major**: Causes frequent friction or high confusion.
- **Minor**: Noticeable friction with workaround.
- **Cosmetic**: Does not block completion.
