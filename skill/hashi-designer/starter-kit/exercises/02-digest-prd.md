# Exercise 2: Digest PRD

Work with your own PRD and technical RFC/architecture documents.

---

## 1. Analyzing User Journey

Prompts to try:

- "map the user journey for the [primary workflow]"
- "what are the critical user journeys in this PRD?"
- "write CUJs for [persona] using this product"
- "where are the handoffs between human and automated steps?"

---

## 2. OOUX Analysis

Prompts to try:

- "do an OOUX analysis of this product"
- "what are the core objects in this system?"
- "map the objects, relationships, and CTAs"
- "what's the primary object users interact with?"

---

## 3. Tech Data Ingestion

Prompts to try:

- "summarize the RFC for design purposes"
- "what technical constraints should I know about?"
- "how does the system architecture affect the UX?"
- "what states can a work item be in?"

---

## 4. UXDR (UX Decision Record)

Prompts to try:

- "create a UXDR for the [key workflow] decision"
- "document the design decision for [approach A] vs [approach B]"
- "what trade-offs are we making with this approach?"

---

## 5. Proposal of UI

Prompts to try:

- "based on this PRD, what screens do we need?"
- "propose a UI structure for this product"
- "what's the minimum viable UI for phase 1?"
- "wireframe the [primary view]"

---

## Output

Save your outputs as you go:
- `user-journeys.md`
- `ooux-analysis.md`
- `uxdr/` folder for decision records
