# PDR-004: Incident Response Dashboard - Wireframe Plan

**Status**: Active
**Date**: 2026-02-21
**Author**: Maya Chen
**Depends on**: [PDR-003.JourneyMap-IncidentResponse](003.26.02.14.PDR-JourneyMap-IncidentResponse.md)

## Problem Statement

The PDR-003 data contracts define 10 stages of incident lifecycle data. The wireframes must render these contracts into a navigable operator interface where the agent assists at every stage. Each story must show realistic fixture data, grayscale-only visual hierarchy, and inline agent chat - so stakeholders can evaluate the flow before any production code is written.

---

## Design Constraints

- **Grayscale only**: Shapes, glyphs, and text weight differentiate status - not color. Use `wireframeTok` CSS custom properties from shared fixtures.
- **Inline styles**: All components use `React.CSSProperties` referencing `wireframeTok`. No external CSS files.
- **CSF3 format**: Storybook Component Story Format 3. Stories export `meta` default and named story constants.
- **CSS Grid layout**: All panel arrangements use CSS Grid. No absolute positioning except the top-level shell frame (nav bar).
- **Responsive within 1440px**: Max-width container. Grid columns collapse at narrower widths but the primary target is 1440px desktop.
- **Inline components**: Each story file defines its own React components. No shared component library imports - only fixture data imports.
- **Agent chat panel**: Present in every story. Right-side panel showing contextual agent messages.
- **Nav bar**: Present in every story. Horizontal tabs across the top using `navTabs` from fixtures.

### Status Glyph Function

```typescript
export function statusGlyphFor(status: string): string {
  const map: Record<string, string> = {
    pass: '\u2713',      // checkmark
    done: '\u2713',
    fail: '\u2717',      // X
    failed: '\u2717',
    running: '\u25CF',   // filled circle
    pending: '\u25CB',   // empty circle
    'rolled-back': '\u21A9', // return arrow
    skipped: '\u2014',   // dash
    critical: '\u25B2',  // triangle up
    high: '\u25B2',
    medium: '\u25C6',    // diamond
    low: '\u25CB',
  };
  return map[status] || '\u25CB';
}
```

## Storybook Conventions

- **Title pattern**: `Wireframes/Incident/{StageName}` where StageName is: AlertCorrelation, Triage, Analysis, Approval, Execution, VerifyClose
- **Layout**: `fullscreen` in all story meta
- **Render pattern**: All stories use `render: () => <ComponentName />` where the component is defined inline in the same file
- **Inline components**: Every visual element is a function component defined in the story file. Only imports from `_incident-fixtures.ts`.
- **State management**: Use `React.useState` for interactive elements (tab selection, list selection, expandable rows). Initial state is set to show the story's intended view.
- **Documentation**: Every story meta includes `parameters: { docs: { description: { component: '...' } } }` with the ASCII wireframe from this PDR
- **No color**: All `background`, `color`, and `border` properties use `wireframeTok.*` values only
- **Grid tokens**: Use `wireframeTok.borderSubtle` for all grid lines and separators
- **Text hierarchy**: `wireframeTok.textPrimary` for labels, `wireframeTok.textSecondary` for metadata, `wireframeTok.textPlaceholder` for empty states
- **Panel backgrounds**: Alternating `wireframeTok.bg` and `wireframeTok.layer01` for visual separation
