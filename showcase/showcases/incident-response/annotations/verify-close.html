What the user is doing

Agent runs verification checks automatically after execution completes. The operator reviews check results, confirms resolution, and closes the incident. A 120-second quiet window confirms no new correlated alerts before closure is permitted.

Screen breakdown

  • Incident summary / Closure metadata (30% left): Final incident card showing severity, duration, root cause, and remediation steps executed. Closure fields include resolver, timestamp, and IC sign-off status for severity-1/2 incidents.
  • Verification checklist / Incident timeline (40% center): Top section lists automated verification checks with pass/fail glyphs. Bottom section shows the full incident timeline from first alert through closure.
  • Agent chat (30% right): Agent reports verification results, confirms the quiet window passed, and prompts the operator to close.

Key interactions

  • Review verification check results - each shows what was tested and the outcome.
  • Click Close Incident to finalize (requires IC sign-off for severity-1/2).
  • Review the full incident timeline from correlation through resolution.

Design decisions

  • Verification is automatic. The agent runs checks without operator prompting - the operator reviews results, not procedures.
  • 120-second quiet window confirms no new correlated alerts before closure is permitted. Recurrence restarts the window.
  • Severity-1/2 incidents require IC sign-off to close. Lower severities close with operator confirmation alone.
  • Full timeline is written to the Splunk audit trail on closure - every action, approval, and verification captured.
