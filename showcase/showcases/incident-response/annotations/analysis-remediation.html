What the user is doing

Agent presents ranked root causes with confidence scores. The operator reviews evidence chains, confirms the most likely cause, and reviews the remediation playbook. Every mutating step in the playbook has a rollback command defined before the operator can request approval.

Screen breakdown

  • Incident context (25% left): Summary card showing incident ID, severity, affected subsystems, and timeline of correlated signals for quick reference.
  • Root cause ranking / Remediation plan (45% center): Top section lists ranked causes with confidence bars and expandable evidence chains. Bottom section shows the remediation playbook with sequential steps, parameters, and risk markers.
  • Agent chat (30% right): Agent walks through its reasoning - which log entries, metrics, and patterns led to each confidence score.

Key interactions

  • Review confidence bars for each candidate root cause.
  • Expand evidence chains to see the supporting log entries and metrics.
  • Click a cause to select it and load the corresponding playbook.
  • Review playbook steps, edit parameters if needed.
  • Click Request Approval to send the plan to the Incident Commander.

Design decisions

  • Confidence bars use filled/empty blocks in grayscale, not color - accessible and readable on any display.
  • High-risk steps are marked with a triangle glyph and require Incident Commander approval before execution.
  • Every mutating step must have a rollback command defined. The playbook is incomplete without reversal paths.
