What the user is doing

Operator sees incoming alerts. The agent ingests run logs and audit events and drift detection results, groups correlated signals across workspaces. Correlation happens automatically - the agent groups related signals within 10 seconds of ingestion, surfacing a single incident from what would otherwise be a wall of disconnected alerts.

Screen breakdown

  • Alert list (25% left): Chronological feed of raw alerts. Each row shows timestamp, source subsystem, and severity badge. Correlated alerts collapse under a group header.
  • Signal detail with correlation table (45% center): Expanded view of the selected alert group. Correlation table maps each signal to its source, timestamp delta, and match reason.
  • Agent chat (30% right): Agent narrates correlation logic - which signals matched, why they grouped, and what subsystems are involved.

Key interactions

  • Click an alert row to expand the correlation group in the center panel.
  • Review the correlation table to verify grouping logic.
  • Confirm grouping to promote the cluster to an incident, or ungroup a mismatched signal back to the raw queue.

Design decisions

  • Three-panel layout keeps the alert queue visible while the operator investigates a specific group - no context switching.
  • Correlation is automatic. The agent groups within 10 seconds; the operator confirms rather than constructs.
  • Ungroup action is always available so the operator retains final authority over what constitutes a single incident.
