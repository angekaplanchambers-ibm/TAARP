What the user is doing

Operator selects an incident from the queue, reviews the triage summary, and accepts auto-assignment. The agent has already classified severity, identified affected subsystems, and recommended an escalation path based on historical pattern match.

Screen breakdown

  • Incident table (30-70% depending on mode): Queue of open incidents. Columns show ID, severity, subsystem, age, and assignee. Table compresses to 30% when a detail panel is open.
  • Detail panel with severity/signals/subsystems (40% center): Triage summary for the selected incident. Shows correlated signal count, severity justification, affected subsystems, and escalation recommendation.
  • Agent chat (30% right): Agent explains the triage rationale - why this severity, which historical incidents matched, and the recommended next step.

Key interactions

  • Click an incident row to open the detail panel; table compresses to make room.
  • Review the escalation recommendation and severity classification.
  • Click Analyze to accept triage and proceed to root cause analysis.

Design decisions

  • Table-to-detail transition collapses the table rather than replacing it, so the operator keeps queue context while triaging.
  • Escalation recommendation is based on historical pattern match - the agent cites similar past incidents, not abstract rules.
  • Single Analyze action moves forward. No ambiguous multi-button toolbar at the triage stage.
