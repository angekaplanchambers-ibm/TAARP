What the user is doing

Incident Commander reviews and approves the remediation plan. A 15-minute approval window prevents the request from blocking indefinitely. The IC reads the plan, confirms high-risk steps, and either approves with a reason or rejects to send the operator back to remediation.

Screen breakdown

  • Approval metadata with countdown (30% left): Shows requester, incident severity, submission time, and a countdown timer for the 15-minute approval window. Escalation target displayed below the timer.
  • Plan review read-only (40% center): The remediation playbook rendered in read-only mode. High-risk steps highlighted with triangle glyphs. Each step shows the command, target subsystem, and rollback path.
  • Agent chat (30% right): Agent summarizes the plan rationale, flags which steps carry risk, and notes any conditions that must hold for the plan to succeed.

Key interactions

  • Review each high-risk step and its rollback command.
  • Enter a reason in the text field (required for audit trail).
  • Check the confirmation checkbox and click Approve, or click Reject to return to remediation.
  • View pre-conditions that must hold before execution begins.

Design decisions

  • Approval checkbox is deliberate friction - the IC must explicitly confirm, preventing accidental approval.
  • 15-minute timer prevents approval from blocking indefinitely. Timeout triggers automatic escalation to Infrastructure Architect.
  • Rejection returns to the remediation step, not the beginning of the workflow. Progress is preserved.
  • Read-only plan view prevents the IC from editing steps inline - changes require a new remediation cycle.
