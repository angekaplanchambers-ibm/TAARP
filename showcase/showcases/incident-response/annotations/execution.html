What the user is doing

Agent executes playbook steps sequentially via terraform CLI. The operator monitors real-time progress as each step runs, validates, and advances. Failure halts execution immediately - the operator chooses whether to retry, rollback, or escalate.

Screen breakdown

  • Execution summary with progress bar (30% left): Shows incident ID, total steps, completed count, and a segmented progress bar. Current step highlighted. Elapsed time and estimated remaining time displayed below.
  • Step list with status glyphs and output (40% center): Each step shows a status glyph (pending, running, passed, failed), the command being executed, and expandable monospace output. The running step auto-expands to show live output.
  • Agent chat (30% right): Agent narrates each step as it executes - what it ran, what the output means, and whether validation passed.

Key interactions

  • Watch steps complete in real time as glyphs update from pending to running to passed.
  • Expand any completed step to review its output.
  • On failure, choose a recovery path: retry the step, rollback all completed steps, or escalate to Infrastructure Architect.

Design decisions

  • Sequential execution with validation between steps. Each step must pass before the next begins.
  • Failure halts immediately. No partial execution of remaining steps.
  • Rollback executes in reverse order of completed steps, using the rollback commands defined in the playbook.
  • Event stream provides monospace output so the operator can verify exactly what ran and what it returned.
